# Workspace

## Overview

pnpm workspace monorepo using TypeScript. Each package manages its own dependencies.

## Stack

- **Monorepo tool**: pnpm workspaces
- **Node.js version**: 24
- **Package manager**: pnpm
- **TypeScript version**: 5.9
- **API framework**: Express 5
- **Database**: PostgreSQL + Drizzle ORM
- **Validation**: Zod (`zod/v4`), `drizzle-zod`
- **API codegen**: Orval (from OpenAPI spec)
- **Build**: esbuild (CJS bundle)

## Key Commands

- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- `pnpm --filter @workspace/api-server run dev` — run API server locally

See the `pnpm-workspace` skill for workspace structure, TypeScript setup, and package details.

## Side project: GhostType Pro (Android keyboard APK)

`ghosttype-pro/` contains a complete native Android Kotlin/Jetpack Compose IME project (separate from the pnpm monorepo). It is built in the cloud via the GitHub Actions workflow at `.github/workflows/android-build.yml` — pushing the repo to GitHub produces downloadable debug + release APK artifacts. Includes 10 modules (IME, Auto-Type, suggestions, themes, fonts, clipboard, macros, multi-language, settings, floating pointer) plus an auto-save sentences feature backed by Room.

New `ApprovalRefreshWorker` (CoroutineWorker, `androidx.work` 2.9) runs every 15 min when the device has internet, calls `ApprovalGate.evaluate(force=true)` to bypass the cache and update the SharedPrefs decision, and on `Blocked` / `NotApproved` results sends a package-scoped broadcast `com.ghosttype.action.APPROVAL_REVOKED`. Useful for chat apps where the user wants to address each message to a specific person without re-typing the name in every line of their source text.

- **Floating sender SEND fires reliably after AutoType line** (`FloatingPointerService.startSend` / `endSend`): `removeView`/`addView` calls were happening on the AutoTypeEngine's IO/coroutine thread → silently throwing `CalledFromWrongThread` and the system never delivered the synthetic touch. Wrapped both in `mainHandler.post {}` so the WindowManager mutation runs on the UI thread. Bumped `AutoTypeEngine.pointerClick()` settle delay 120 → 200 ms and hide duration 450 → 500 ms to give the dispatch full headroom. "Lock = invisible" preserved (alpha 0 + `View.INVISIBLE`).
- **Suggestions now context-aware Roman Urdu + bigrams** (`SuggestionEngine.kt` rewrite): added `BUILTIN_UR_LATIN` (~180 common Roman Urdu words: kya, hai, nahi, theek, accha, kaise, ho, kar, mein, etc.) and a curated bigram table (UR + EN: "kya"→"hai", "ap"→"kaise", "how"→"are", etc.) seeded into the Room `dictionary_words` table on first run. Bigrams are stored as rows where `word = next` and `nextWord = previous` (matches the existing DAO query `WHERE nextWord = :prevWord`). `suggest(prefix, prevWord)` now returns next-word predictions when prefix is empty, and bigram-weighted completions when typing. `learnSentence` walks pairs and persists every `(prev, next)` it sees, so the keyboard learns the user's actual phrasing over time. `GhostTypeIMEService.updateSuggestions` already passes `prevWord` from text-before-cursor — no change needed there.
- **Custom theme background no longer changes keyboard size** (`KeyboardView.applyKeyboardBackground` → `NoIntrinsicBitmapDrawable`): replaced stock `BitmapDrawable` (which reports the bitmap's pixel width/height as intrinsic + minimum size, causing the parent LinearLayout to grow when a tall image was picked) with an inner `NoIntrinsicBitmapDrawable` that returns `-1` from `getIntrinsicWidth/Height` and `0` from `getMinimumWidth/Height`. The image still renders (anchored bottom + scaled via gravity flags from v1.3) but contributes zero size to layout — keyboard height is now driven solely by the user's "Key height" slider regardless of image dimensions.
- **AutoTyper + sender PAUSE on app close/minimize, not stop** (`MainActivity.onDestroy` / `onStop` / `onResume`; `AutoTypeScreen` callbacks): pressing Start now calls `moveTaskToBack(true)` from inside `MainActivity.onStartAutoType` (and the new `onResumeAutoType`) so AutoTypeScreen no longer initiates the move itself — this lets `MainActivity` record `lastStartAtMs` immediately before backgrounding. `onStop()` checks if the system-triggered pause happened within 2 s of `lastStartAtMs` (the Start-induced minimize) — if so, no pause; otherwise calls `AutoTypeEngine.pause()` so a manually-minimized app freezes the typer mid-line. `onDestroy` (swipe from Recents) and `onResume` (re-foreground) likewise call `pause()` instead of `stop()`. Foreground service stays alive across pause because `state.running` remains true. Renamed compose param `onResume` → `onResumeAutoType` to avoid colliding with the Activity lifecycle callback.

Recent improvements (v1.3.1 — custom font fix):
- **Custom .ttf fonts now actually apply** (`FontManager.loadCustomTypeface`, `importFont`, `setActive`): the previous `Typeface.createFromFile(path)` call silently fell back to `Typeface.DEFAULT` on many OEM ROMs (Xiaomi MIUI, Vivo / Oppo ColorOS, certain Samsung builds) for valid `.ttf` files containing DSIG signature tables, GSUB extensions, or variable-font axes — so the user picked a custom font and saw Roboto. Fixed with a 3-tier loader: (1) `Typeface.Builder(file).build()` on API 26+ which uses Android's modern Minikin loader and accepts a much broader font format set, (2) fallback to `createFromFile`, (3) explicit detection of "loader returned `Typeface.DEFAULT`" treated as failure. `importFont` now validates the file BEFORE listing it — invalid fonts are deleted and the user gets a clear toast ("Couldn't load <name>. The .ttf may be unsupported"). `setActive` now invalidates the typeface cache before writing the pref so the SharedPreferences listener can never read a stale cached typeface for the new path. The on-keyboard `Aa` picker preview shares the same loader so what you see in the picker is what lands on the keys.
